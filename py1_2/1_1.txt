1) Программа "Telegram". В ней должен быть класс пользователя, содержащей различную информацию о нём самом (его имя, номер телефона, имя пользователя, дата рождения и другая личная информация).
Также должен иметься список контактов, список чатов, групп, каналов (чат и группа являются отдельными классами).
Чат содержит в себе список из двух пользователей, их ники в данном чате, историю сообщений (включая различные ссылки, изображения и остальное).
Группа и канал реализуются примерно аналогичным образом, добавляя свою конкретику (например, большее количество пользователей).
Также должен быть класс "меню", в котором используется набор кнопок-опций, центральное окно для чатов, различные опциональные панели (например, список чатов/групп/каналов, в которых состоит пользователь).
2) Игра HoMM3. В игре должен быть класс боевой единицы, с набором различных характеристик (атака, защита, здоровье и т.п.), класс заклинаний (у каждого должна быть стоимость, имя, список школ магии,
к которым относится данное заклинание). Класс "Книга магии" будет содержать список всех заклинаний, которые герой знает на данный момент. Сам же класс "Герой" будет содержать в себе множество объектов
других классов, в частности: объекты класса "Навык", с уровнем владения, список объектов класса "Артефакт", набор из не более чем семи объектов класса "юнит", с дополнением их количества в ячейке
(которая реализована отдельным классом) и т.п.
